# 日报内容契约（v2）

## 目标读者

日报首先是一份给普通但关心科技、商业与公共议题读者的“信息早餐”，目标是在 5—10 分钟内完成扫读。

- 读者先用 30 秒结论判断是否继续。
- 想多了解时再展开完整摘要、核心观点、关键事实与分歧限制。
- 需要研究底稿时回到完整字幕与证据产物，不把所有研究内容摊在日报里。

## 模型职责边界

模型只填写 JSON 内容，不负责 Markdown、XML 或飞书排版。

程序固定负责：

- 标题层级、callout、grid、表格、颜色和链接。
- 中英文与数字之间的空格。
- 字数、条目数量和字段完整性。
- 引用类型、证据 ID 与数字证据校验。
- 输出失败后的修复重试和最终发布门禁。

模型不得输出 Markdown 标题、表格、引用符号、XML 标签或代码围栏。

### 飞书格式边界

以下内容全部由程序固定生成，模型不得在任何字段里手写或暗示：

- 文档标题、一级 / 二级 / 三级标题层级。
- `今日信息地图` 的 mermaid / whiteboard。
- `3 分钟速览`、栏目说明、单篇元信息和 30 秒结论 callout。
- callout 颜色、emoji、边框色。
- grid 双栏 / 三栏布局。
- 关键事实表格。
- 引用块、链接、加粗、字体颜色、星级显示。
- “全部更新”下固定五大分类，以及无更新分类的“今日无新增”。

模型字段里不得出现 Markdown 标题、表格、XML、HTML、mermaid、callout、grid、颜色、字号、加粗标记或代码块。格式错了不是“更好看”，而是合同失败。

## 内容规则

### 总体防幻觉规则

- 字幕或证据中没有明确出现的人名、公司、年份、金额、比例、职位、政策、产品状态，一律不要写。
- 不要根据标题、常识、新闻背景或模型自己的知识补充事实。
- 不要为了让摘要“更完整”补节目没有讲的上下文。
- 英文内容可以翻译和概括，但必须保留原意；不确定的术语宁可保留英文。
- 字幕质量差时，只写能确定的部分，并在“分歧与限制”说明信息限制，不编造缺失内容。
- 同一个事实不要在“完整摘要 / 核心要点 / 关键事实”中机械重复。

### 30 秒结论

- 真正控制在 30 个非空白字符以内。
- 只表达一个主题和一个结果，不承担背景说明。
- 与“为什么值得看”分工：前者说内容，后者说和读者的关系。
- 不写“本期讨论了……”“嘉宾分享了……”这类空泛导语。

### 完整摘要

- 30 分钟以下内容保留 2—4 段；30 分钟以上保留 3—5 段；60 分钟以上或高密度内容保留 4—6 段。
- 每段只处理一个议题，不超过 150 个非空白字符。
- 不按时间顺序复述，不重复核心要点。

### 核心要点

- 按时长与密度保留 3—7 条；长节目不得强行压成三条。
- 每条必须是判断或因果关系，不写“讨论了某话题”。
- 每条必须引用真实证据 ID。
- 可以写机制、趋势、权衡、约束或可迁移框架；不要写章节目录式复述。

### 关键事实

- 使用统一的“标签 / 值 / 上下文”结构，由程序渲染为两列表格。
- 数字必须能在引用的证据片段中找到；无法校验时删除，不猜测。
- 没有可靠事实时允许留空，不为了填表制造内容。
- 只放可核对事实、数字、年份、金额、比例、机构动作或明确案例；不要放观点或推测。

### 你可以怎么用

- 替代旧版“值得后续整理的问题”。
- 写普通读者能执行的阅读方法、判断方式或行动提示。
- 禁止把作者的研究 Todo List 伪装成读者思考题。
- 不能写问句，不能写“继续关注”这种无动作泛话。

### 嘉宾与机构

- 只写证据中明确出现的人物、机构、角色。
- 不知道职位、头衔或公司关系时不要补。
- 不把主持人、广告主、片头节目名误当成核心嘉宾。

### 分歧与限制

- 写对立观点、适用边界、风险、限制或未决问题。
- 不写泛泛的“仍需观察”。
- 如果字幕质量差、信息缺口大或讲者只给单方观点，应在这里说明。

### 引用

- 只有确实值得记忆的表达才使用。
- `verbatim` 必须能在原字幕证据中逐字匹配。
- 翻译或重构内容必须标为 `translated` / `paraphrase`，飞书端自动显示“意译”。

## 排版规则

- 中英文、中文与阿拉伯数字之间保留一个空格，如 `AI 资本开支`、`KOSPI 大涨`。
- 加粗只用于短标签和短结论，不加粗整段。
- 每篇使用同一信息顺序，关键事实统一为表格。
- “全部更新”下固定保留五大分类；无更新的分类明确显示“今日无新增”。
- “嘉宾与机构 / 完整摘要 · 深读 / 核心要点 / 你可以怎么用 / 分歧与限制”只使用正文加粗，不使用标题，避免污染飞书目录。
- 分隔线不用于逐篇机械分隔；栏目使用灰色 callout 形成视觉区隔。
- 开头提供指标分栏、3 分钟速览和一张今日信息地图。
