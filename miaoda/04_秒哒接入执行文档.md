# 秒哒接入执行文档

## 1. 目标

本仓库只做 `podcast-distill` 的秒哒消费层。

最终链路：

```text
GitHub Actions
  -> POST /daily-collect
  -> GET /daily-collect/{job_id}
  -> 下载 daily_items.json / manifest.json / subtitles_bundle.zip
  -> 解压 subtitles/
  -> 校验 text_chars / sha256 / coverage_ratio
  -> 基于完整逐字稿生成日报
  -> 发布飞书
  -> 上传 artifacts
```

核心原则：

```text
秒哒负责媒体解析与转写。
GitHub Actions 负责调度、校验、生成日报和发布。
```

## 2. 秒哒承担的职责

秒哒必须完成：

- 读取 `sources_profile=podcast-distill-default`。
- 展开 YouTube playlist。
- 展开 YouTube channel / uploads。
- 识别日报窗口内真实更新的视频。
- 展开小宇宙 podcast。
- 识别日报窗口内真实更新的 episode。
- 解析单条小宇宙 episode 的真实 `audio_url`。
- YouTube 优先提取官方字幕。
- 没有官方字幕时提取平台自动字幕。
- 没有字幕时在秒哒服务器侧 ASR。
- 小宇宙音频在秒哒服务器侧 ASR。
- 输出完整真实逐字稿。
- 输出 `daily_items.json`。
- 输出 `manifest.json`。
- 输出 `subtitles_bundle.zip`。
- 缓存单条媒体结果，避免重复转写。
- 失败时写入结构化 `errors[]`，不能静默跳过。

秒哒不能做：

- 用大模型根据标题、简介、章节、评论补写逐字稿。
- 用摘要冒充逐字稿。
- 转写不完整却返回 `success`。
- 把 playlist 当作单条视频返回。
- 把小宇宙 podcast 主页当作单条 episode 返回。

## 3. GitHub Actions 承担的职责

GitHub Actions 必须完成：

- 定时触发。
- 手动触发并支持 `date=YYYY-MM-DD`。
- 明确传入 `window_start` 和 `window_end`。
- 调用秒哒 `/daily-collect`。
- 轮询任务状态。
- 下载三类产物。
- 解压 `subtitles_bundle.zip`。
- 校验产物。
- 校验通过后运行日报生成。
- 日报生成通过后发布飞书。
- 成功或失败都上传 artifacts。

GitHub Actions 不能做：

- 不跑 ASR。
- 不安装 Whisper / whisper.cpp。
- 不调用 `yt-dlp`。
- 不下载 YouTube 音频。
- 不下载小宇宙音频。
- 不直接抓 YouTube 字幕。
- 不直接解析小宇宙音频链接。

## 4. 主 API

公网 Base URL：

```text
https://backend.appmiaoda.com/projects/supabase330558257305272320/functions/v1
```

健康检查：

```http
GET /health
```

创建每日采集任务：

```http
POST /daily-collect
Authorization: Bearer <token>
Content-Type: application/json
```

请求体：

```json
{
  "date": "2026-06-30",
  "window_start": "2026-06-29T06:00:00+08:00",
  "window_end": "2026-06-30T06:00:00+08:00",
  "sources_profile": "podcast-distill-default",
  "require_transcripts": true,
  "allow_asr": true
}
```

返回：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "queued",
  "created_at": "2026-07-01T03:28:05Z"
}
```

轮询：

```http
GET /daily-collect/{job_id}
Authorization: Bearer <token>
```

成功状态必须包含文件入口：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "success",
  "date": "2026-06-30",
  "item_count": 8,
  "success_count": 8,
  "failure_count": 0,
  "files": {
    "daily_items_json": "/daily-collect/daily_2026-06-30_xxx/files/daily_items.json",
    "manifest_json": "/daily-collect/daily_2026-06-30_xxx/files/manifest.json",
    "subtitles_bundle_zip": "/daily-collect/daily_2026-06-30_xxx/files/subtitles_bundle.zip"
  }
}
```

如果没有 `files` 字段，GitHub Actions 会按固定路径下载：

```text
/daily-collect/{job_id}/files/daily_items.json
/daily-collect/{job_id}/files/manifest.json
/daily-collect/{job_id}/files/subtitles_bundle.zip
```

## 5. 产物格式

`daily_items.json` 必须是数组。每个条目至少包含：

```json
{
  "platform": "xiaoyuzhou",
  "category": "新闻 / 时评 / 全球议题",
  "source_name": "晚点聊 LateTalk",
  "source_url": "https://www.xiaoyuzhoufm.com/podcast/...",
  "title": "节目标题",
  "original_title": "节目标题",
  "url": "https://www.xiaoyuzhoufm.com/episode/...",
  "published_at": "2026-06-29T03:30:00+00:00",
  "duration": 6830,
  "description": "...",
  "chapters": []
}
```

`manifest.json` 必须是对象：

```json
{
  "status": "success",
  "date": "2026-06-30",
  "window_start": "2026-06-29T06:00:00+08:00",
  "window_end": "2026-06-30T06:00:00+08:00",
  "source_count": 41,
  "item_count": 8,
  "success_count": 8,
  "failure_count": 0,
  "counts_by_platform": {
    "youtube": 5,
    "xiaoyuzhou": 3
  },
  "errors": []
}
```

`subtitles_bundle.zip` 解压后必须是：

```text
subtitles/
  xiaoyuzhou_6a40b830ffcf45052c805d5f.json
  xiaoyuzhou_6a40b830ffcf45052c805d5f.txt
  xiaoyuzhou_6a40b830ffcf45052c805d5f.srt
  youtube_XXXXXXXXXXX.json
  youtube_XXXXXXXXXXX.txt
  youtube_XXXXXXXXXXX.vtt
```

每个 `subtitles/*.json` 必须至少包含：

```json
{
  "url": "原始媒体链接",
  "text": "xxx.txt",
  "text_chars": 12345,
  "sha256": "xxx.txt 的 UTF-8 sha256",
  "duration_seconds": 1234,
  "last_timestamp_seconds": 1230,
  "coverage_ratio": 0.99,
  "source": "official_caption"
}
```

`source` 只接受：

```text
official_caption
auto_caption
asr
```

如果秒哒内部想保留命名空间，也可返回 `miaoda:asr`，GitHub Actions 会取冒号后的方法名校验。

## 6. GitHub Actions 校验闸口

正式 workflow 使用：

```text
scripts/miaoda_collect_daily.py
scripts/validate_miaoda_bundle.py
scripts/generate_daily_report.py
scripts/publish_feishu.py
```

`validate_miaoda_bundle.py` 会校验：

- `manifest.status` 必须是 `success` 或 `complete`。
- `manifest.failure_count` 必须为 0。
- `manifest.errors` / `manifest.failures` 必须为空。
- zip 必须包含 `subtitles/` 根目录。
- 每个 `subtitles/*.json` 必须有 `url` 和 `text`。
- `text` 指向的 txt 文件必须存在。
- txt 必须是 UTF-8。
- 本地复算 `sha256` 必须一致。
- `text_chars` 必须大于 0，且等于本地解码文本长度。
- 必须有 `duration_seconds`。
- 必须有 `last_timestamp_seconds`。
- 必须有 `coverage_ratio`。
- 5 分钟以上内容 `coverage_ratio >= 0.95`。
- 必须有 `.srt` 或 `.vtt`。
- 5 分钟以上 `daily_items` 条目必须能按 URL 匹配到字幕元数据。
- YouTube `daily_items` 不能仍是 playlist/channel URL。

只要校验失败，就不会进入飞书发布步骤。

## 7. 2026-06-30 验收

手动触发 workflow：

```text
date = 2026-06-30
```

窗口必须是：

```text
2026-06-29T06:00:00+08:00
2026-06-30T06:00:00+08:00
```

必须包含并转写成功：

```text
https://www.xiaoyuzhoufm.com/episode/6a41f1212e335a35a80b0159
https://www.xiaoyuzhoufm.com/episode/6a40b830ffcf45052c805d5f
https://www.xiaoyuzhoufm.com/episode/6a412def9d2f5743683f320e
```

workflow 会在 `REPORT_DATE=2026-06-30` 时自动把这 3 个 URL 加入 `--expected-url` 校验。

成功 artifacts 应包含：

```text
reports/daily_items_2026-06-30.json
reports/daily_items_2026-06-30.manifest.json
reports/miaoda_collect_2026-06-30.status.json
reports/miaoda_validation_2026-06-30.json
subtitles_bundle.zip
subtitles/
reports/daily-2026-06-30.md
reports/daily-2026-06-30.json
```
