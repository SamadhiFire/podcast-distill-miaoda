# 秒哒媒体解析与转写服务需求文档

## 1. 核心结论

最终方案必须是：

```text
GitHub Actions 只负责调度、下载、校验、生成日报、写飞书。
秒哒服务负责 YouTube / 小宇宙的解析、字幕提取、音频下载、ASR、缓存和结果打包。
```

也就是说，GitHub Actions 不应该跑 ASR，不应该安装 whisper，不应该下载音频，也不应该用 `yt-dlp` 高频抓 YouTube 字幕或音频。

秒哒必须直接输出日报生成所需的全部文本文件：

```text
daily_items.json
manifest.json
subtitles_bundle.zip
```

其中 `subtitles_bundle.zip` 必须能直接解压成 `podcast-distill` 当前日报脚本可读取的 `subtitles/` 目录。

## 2. 背景

`podcast-distill` 现在用于生成每日播客/视频日报，并写入飞书。现有痛点：

- 本机跑小宇宙长音频 ASR 太慢。
- GitHub Actions 跑 `yt-dlp`、字幕抓取、音频下载或 ASR 容易慢、超时、触发 YouTube 风控。
- 日报必须基于完整真实逐字稿，不能用标题、简介、章节或大模型补写。
- 自动化必须无人值守，失败时不能把不完整日报写入飞书。

因此需要秒哒提供远程“媒体解析与转写服务”。

## 3. 技术栈建议

这些依赖都部署在秒哒服务器侧，不部署在 GitHub Actions 侧。

- 后端：Python 3.11+、FastAPI、Pydantic、Uvicorn/Gunicorn
- 异步任务：Redis + RQ/Celery/Arq
- 存储：SQLite 起步，后续可换 PostgreSQL；音频和结果包可用本地磁盘或对象存储
- 音频处理：`ffmpeg`
- YouTube 解析：`yt-dlp`
- YouTube 官方元数据：YouTube Data API v3
- YouTube 字幕备选：`youtube-transcript-api`
- ASR：优先 `faster-whisper`，无 GPU 时可用 `whisper.cpp`

推荐链接：

- `yt-dlp`: https://github.com/yt-dlp/yt-dlp
- YouTube Data API v3: https://developers.google.com/youtube/v3
- `youtube-transcript-api`: https://github.com/jdepoix/youtube-transcript-api
- `faster-whisper`: https://github.com/SYSTRAN/faster-whisper
- `whisper.cpp`: https://github.com/ggml-org/whisper.cpp
- OpenAI Whisper: https://github.com/openai/whisper

## 4. 必须支持的平台

第一优先级：

- 小宇宙 episode URL
- 小宇宙 podcast URL 的节目列表
- YouTube video URL
- YouTube playlist URL
- YouTube channel uploads

小宇宙必须解析真实音频地址，例如：

- `episode.enclosure.url`
- `episode.media.source.url`
- 页面内嵌 JSON 中的其他真实音频 URL

YouTube 必须优先使用官方字幕或自动字幕；没有字幕时才由秒哒服务器侧 ASR。

## 5. 真实性原则

逐字稿只能来自：

- 官方字幕
- 平台自动字幕
- 音频 ASR

禁止：

- 用大模型根据标题、简介、章节、评论补写逐字稿
- 用摘要冒充逐字稿
- 转写不完整却返回 `success`
- 静默跳过失败条目

允许：

- 标准化换行
- 去掉重复空行
- 从 SRT/VTT 中导出纯文本
- 轻微清洗格式噪声

不允许改变原意。

## 6. 主接口：每日采集

GitHub Actions 最终只应调用这个接口完成当天所有媒体解析和转写。

```http
POST /v1/daily/collect
Authorization: Bearer <token>
Content-Type: application/json
```

请求示例：

```json
{
  "date": "2026-06-30",
  "timezone": "Asia/Shanghai",
  "window_start": "2026-06-29T06:00:00+08:00",
  "window_end": "2026-06-30T06:00:00+08:00",
  "sources_profile": "podcast-distill-default",
  "require_transcripts": true,
  "allow_asr": true
}
```

说明：

- `date` 是日报日期。
- `window_start` 和 `window_end` 必须由调用方明确传入，避免相对日期误判。
- `sources_profile` 推荐由秒哒服务器保存源配置，GitHub Actions 不再上传大段源配置。
- 也可以扩展支持 `sources` 数组，但正式自动化建议用 `sources_profile`。
- `require_transcripts=true` 时，任何 5 分钟以上条目没有字幕/转写，都不能返回成功。

创建任务返回：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "queued",
  "created_at": "2026-06-30T06:16:00+08:00"
}
```

查询任务：

```http
GET /v1/daily/collect/{job_id}
Authorization: Bearer <token>
```

进行中：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "running",
  "stage": "transcribing",
  "progress": {
    "finished_items": 5,
    "total_items": 8,
    "message": "transcribing xiaoyuzhou episode"
  }
}
```

成功：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "success",
  "date": "2026-06-30",
  "window_start": "2026-06-29T06:00:00+08:00",
  "window_end": "2026-06-30T06:00:00+08:00",
  "item_count": 8,
  "files": {
    "daily_items_json": "/v1/daily/collect/daily_2026-06-30_xxx/files/daily_items.json",
    "manifest_json": "/v1/daily/collect/daily_2026-06-30_xxx/files/manifest.json",
    "subtitles_bundle_zip": "/v1/daily/collect/daily_2026-06-30_xxx/files/subtitles_bundle.zip"
  }
}
```

失败：

```json
{
  "job_id": "daily_2026-06-30_xxx",
  "status": "failed",
  "error_type": "missing_required_transcripts",
  "error_message": "2 required items have no complete transcript",
  "retryable": true,
  "failed_items": [
    {
      "url": "https://www.xiaoyuzhoufm.com/episode/...",
      "error_type": "asr_failed",
      "error_message": "coverage_ratio below threshold"
    }
  ]
}
```

## 7. 产物格式

### 7.1 `daily_items.json`

必须是数组，字段兼容当前 `podcast-distill`：

```json
[
  {
    "platform": "xiaoyuzhou",
    "category": "新闻 / 时评 / 全球议题",
    "source_name": "晚点聊 LateTalk",
    "source_url": "https://www.xiaoyuzhoufm.com/podcast/61933ace1b4320461e91fd55",
    "title": "170: 【具身季报 26Q2】世界模型大风不停，和不想被贴标签的人",
    "original_title": "170: 【具身季报 26Q2】世界模型大风不停，和不想被贴标签的人",
    "url": "https://www.xiaoyuzhoufm.com/episode/6a40b830ffcf45052c805d5f",
    "published_at": "2026-06-29T03:30:00+00:00",
    "duration": 6830,
    "description": "...",
    "chapters": []
  }
]
```

时间窗口必须严格等于：

```text
日报日期 D = 北京时间 D-1 06:00:00 到 D 06:00:00
```

例如：

```text
2026-06-30 日报 = 2026-06-29 06:00:00+08:00 到 2026-06-30 06:00:00+08:00
```

### 7.2 `subtitles_bundle.zip`

这个 zip 是 GitHub Actions 的关键输入。它必须可以直接解压到 `subtitles/` 目录。

推荐结构：

```text
subtitles/
  xiaoyuzhou_6a40b830ffcf45052c805d5f.json
  xiaoyuzhou_6a40b830ffcf45052c805d5f.txt
  xiaoyuzhou_6a40b830ffcf45052c805d5f.srt
  youtube_YNx84aaPJz8.json
  youtube_YNx84aaPJz8.txt
  youtube_YNx84aaPJz8.vtt
```

每条 `.json` 必须至少包含：

```json
{
  "platform": "xiaoyuzhou",
  "media_id": "6a40b830ffcf45052c805d5f",
  "title": "170: 【具身季报 26Q2】世界模型大风不停，和不想被贴标签的人",
  "url": "https://www.xiaoyuzhoufm.com/episode/6a40b830ffcf45052c805d5f",
  "language": "zh",
  "source": "miaoda:asr",
  "subtitle_srt": "xiaoyuzhou_6a40b830ffcf45052c805d5f.srt",
  "text": "xiaoyuzhou_6a40b830ffcf45052c805d5f.txt",
  "text_chars": 123456,
  "sha256": "...",
  "duration_seconds": 6830,
  "last_timestamp_seconds": 6828.1,
  "coverage_ratio": 0.9997
}
```

当前 `scripts/generate_daily_report.py` 只强依赖 `url` 和 `text` 两个字段，但 `text_chars`、`sha256`、`coverage_ratio` 必须提供给 GitHub Actions 校验。

### 7.3 `manifest.json`

必须说明整体采集和转写状态：

```json
{
  "status": "success",
  "date": "2026-06-30",
  "window_start": "2026-06-29T06:00:00+08:00",
  "window_end": "2026-06-30T06:00:00+08:00",
  "source_count": 41,
  "item_count": 8,
  "success_count": 8,
  "failure_count": 0,
  "counts_by_platform": {
    "youtube": 5,
    "xiaoyuzhou": 3
  },
  "errors": []
}
```

如果有失败，不能静默跳过，必须列在 `errors` 中。

## 8. 单条媒体接口

单条接口用于调试、补录和人工重跑，不是 GitHub Actions 的主路径。

```http
POST /v1/media/extract
GET /v1/media/extract/{job_id}
GET /v1/media/extract/{job_id}/files/bundle.zip
GET /v1/media/extract/by-url?url=<url>
```

单条成功结果也必须包含：

- `transcript_text`
- `segments`
- `text_chars`
- `sha256`
- `duration_seconds`
- `last_timestamp_seconds`
- `coverage_ratio`
- `source_method`
- `files.transcript_txt`
- `files.transcript_srt` 或 `files.transcript_vtt`

## 9. 完整度验收标准

只有满足以下条件，条目才允许标记成功：

- `transcript.txt` 存在
- `text_chars > 0`
- `sha256` 存在，且按 `transcript.txt` UTF-8 bytes 本地复算一致
- `duration_seconds` 存在
- `last_timestamp_seconds` 存在
- `coverage_ratio >= 0.95`，短音频建议 `>= 0.98`
- 有 `segments` 或 SRT/VTT
- 标明 `source_method=official_caption`、`auto_caption` 或 `asr`

`require_transcripts=true` 时，只要有任何 5 分钟以上条目不满足上述条件，`/v1/daily/collect` 必须返回 `failed` 或 `partial_failed`，不能返回成功。

## 10. 幂等与缓存

必须实现：

- `normalized_url`
- `idempotency_key = sha256(platform + normalized_url + language + need_transcript)`
- 同一 URL 重复提交时直接返回已有成功结果，不重复 ASR
- 如果 `audio_url` 或字幕版本变化，应创建新版本结果

建议缓存：

- metadata cache
- caption cache
- audio cache
- asr result cache
- daily collect result cache

## 11. GitHub Actions 接入要求

GitHub Actions 只需要这些秒哒 secret：

```text
MIAODA_API_BASE
MIAODA_API_TOKEN
```

秒哒服务完成所有 YouTube / 小宇宙解析后，Actions 下载：

```text
reports/daily_items_YYYY-MM-DD.json
reports/daily_items_YYYY-MM-DD.manifest.json
subtitles_bundle.zip -> 解压为 subtitles/
```

然后 Actions 执行：

```bash
python scripts/generate_daily_report.py \
  --date "$REPORT_DATE" \
  --items-json "reports/daily_items_${REPORT_DATE}.json" \
  --subtitles-dir subtitles \
  --require-transcripts \
  --llm-policy required \
  --output "reports/daily-${REPORT_DATE}.md"
```

注意：

- Actions 不跑 ASR。
- Actions 不调用 `extract_subtitles.py`。
- Actions 不安装 whisper。
- Actions 不下载音频。
- Actions 不使用 `YTDLP_COOKIES_B64` 作为正式依赖。

## 12. 2026-06-30 验收用例

必须能通过 `/v1/daily/collect` 跑通这一天：

```text
date = 2026-06-30
window_start = 2026-06-29T06:00:00+08:00
window_end = 2026-06-30T06:00:00+08:00
```

尤其必须包含并转写成功这 3 条小宇宙：

```text
https://www.xiaoyuzhoufm.com/episode/6a41f1212e335a35a80b0159
https://www.xiaoyuzhoufm.com/episode/6a40b830ffcf45052c805d5f
https://www.xiaoyuzhoufm.com/episode/6a412def9d2f5743683f320e
```

验收标准：

- `daily_items.json` 中只包含指定时间窗口内内容
- `subtitles_bundle.zip` 中每个 5 分钟以上条目都有 `.json + .txt + .srt/.vtt`
- 3 条小宇宙都有真实完整逐字稿
- 3 条小宇宙都有 `text_chars`
- 3 条小宇宙都有 `sha256`
- 3 条小宇宙 `coverage_ratio >= 0.95`
- GitHub Actions 下载产物后能成功运行 `generate_daily_report.py --require-transcripts`

## 13. 不做的事情

秒哒第一阶段不需要做：

- 日报总结生成
- 飞书写入
- 复杂前端编辑器
- 多用户权限系统
- 绕过 YouTube 登录、CAPTCHA、付费或私有内容限制

秒哒只负责：采集、解析、字幕/ASR、缓存、打包、返回可校验结果。
